Overview

Welkom bij deze lessenserie over Node.js! In deze serie zullen we je kennis laten maken met Node.js, een populaire JavaScript-runtime die vaak wordt gebruikt voor het bouwen van server-side webapplicaties. We beginnen met een introductie tot Node.js en de voordelen ervan ten opzichte van traditionele server-side technologieën. Vervolgens zullen we dieper ingaan op het gebruik van Node.js en de belangrijkste concepten en functies die je nodig hebt om krachtige en schaalbare webapplicaties te bouwen. We zullen ook verschillende hulpmiddelen en frameworks onderzoeken die kunnen worden gebruikt in combinatie met Node.js, zoals Express en Socket.IO. Of je nu nieuw bent in de wereld van webontwikkeling of al enige ervaring hebt met JavaScript, deze lessenserie zal je helpen om je kennis van Node.js te vergroten en je vaardigheden te ontwikkelen als webontwikkelaar. Laten we aan de slag gaan!

Internal Modules

TAGS: EmcaScriptModule, Console, OS, file system, HTTP, Documentation

Modules zijn subsystemen dat je kan gebruiken.

Er zijn veel interne modules beschikbaar in Node.js. Hieronder vind je een paar voorbeelden van populaire en nuttige interne modules:

  1. fs - Dit is de File System module van Node.js. Het biedt methoden om te werken met het bestandssysteem van de computer, zoals het lezen en schrijven van bestanden.
  2. path - Dit is een hulpprogramma-module die gebruikt wordt voor het manipuleren van bestandspaden. Het biedt methoden om paden samen te voegen, te normaliseren en om te zetten tussen verschillende notaties.
  3. os - De OS-module biedt toegang tot informatie over het besturingssysteem waarop Node.js wordt uitgevoerd, zoals de hoeveelheid beschikbare geheugen, de processorarchitectuur en het besturingssysteemtype.
  4. http - De HTTP-module is bedoeld voor het maken van HTTP-servers en clients. Het biedt de mogelijkheid om HTTP-verzoeken te verzenden en te ontvangen, en om HTTP-servers te maken die kunnen reageren op verzoeken van clients.
  5. util - Dit is een hulpprogramma-module die een reeks handige hulpprogramma's biedt, zoals het omzetten van objecten naar JSON en vice versa, het formatteren van strings en het erven van klassen.

Dit zijn slechts enkele voorbeelden van de interessante interne modules van Node.js. Node.js biedt veel meer modules die kunnen worden gebruikt om verschillende taken te automatiseren en de ontwikkeling van applicaties te vereenvoudigen. De volledige lijst aan interne modules kan je vinden in de documentatie van NodeJS:

Er zijn twee manieren om modules in jou script in te laden. met require of met import. Hiervoor moeten we eerst een keuze maken tussen CommonJS en ModuleJS.

Het verschil tussen CommonJS en ModuleJS

CommonJS en ModuleJS zijn beide systemen voor het structureren van JavaScript-code in modules, maar er zijn enkele belangrijke verschillen tussen de twee:

CommonJS gebruikt een 'require' functie om stukjes code op te halen en een 'module.exports' object om stukjes code te delen. Het laadt stukjes code op het moment dat ze nodig zijn.

ModuleJS daarentegen gebruikt 'import' en 'export' om stukjes code op te halen en te delen. Het laadt stukjes code voorafgaand aan het uitvoeren van de code.

In het algemeen wordt ModuleJS gezien als de modernere en standaard manier om stukjes code te delen in JavaScript, terwijl CommonJS voornamelijk gebruikt wordt op oudere projecten of op servers zoals Node.js.

FileSystem - non-blocking vs blocking

// Blocking
const fs = require('fs');
const data = fs.readFileSync('/file.md'); // blocks here until file is read
console.log(data);
moreWork(); // will run after console.log

// Non-blocking
const fs = require('fs');
fs.readFile('/file.md', (err, data) => {
if (err) throw err;
console.log(data);
});
moreWork(); // will run before console.log

Intermezzo: Landing page

TAGS: Public folder, templates

Een landing page voor een applicatie is een webpagina die is ontworpen om bezoekers aan te trekken en hen te overtuigen om de mobiele of desktop applicatie te downloaden en te gebruiken. Het doel van deze pagina is om potentiële gebruikers te informeren over de functionaliteit, kenmerken en voordelen van de applicatie, evenals om hen te overtuigen om de applicatie te downloaden en te installeren.

Een goede landing page voor een applicatie moet duidelijk en beknopt zijn, met een aantrekkelijke titel, beschrijving van de belangrijkste kenmerken van de applicatie, screenshots of video's van de applicatie in actie, en sociale bewijsvoering in de vorm van gebruikersrecensies en beoordelingen.

Opdracht: Bedenken van een geweldige applicatie

  1. Maak in de root van het project een nieuwe map aan met de naam "public".
  2. Ga op zoek naar een (gratis) landing page template en download de source code. Er zijn verschillende templates beschikbaar die je kunt gebruiken, kies er één die het beste bij jouw app idee past.
  3. Pak het gedownloade template uit en kopieer de inhoud naar de "public" map van jouw project.
  4. Open het index.html-bestand van het template in jouw teksteditor. Pas de inhoud aan met jouw eigen app idee. Je kunt bijvoorbeeld de kopjes, paragrafen en afbeeldingen vervangen door inhoud die beter bij jouw app past.
  5. Als je wijzigingen hebt aangebracht in de HTML van het template, open dan ook het bijbehorende CSS-bestand en voeg eventueel jouw eigen stijlen toe.
  6. Open het index.html-bestand in jouw browser. Je zou nu jouw aangepaste landing page met jouw eigen app idee moeten zien.
  7. Maak een screenshot van jouw landing page.

External Modules

TAGS: npm, package.json, express, nodemon

Er zijn duizenden externe modules beschikbaar voor Node.js in de vorm van pakketten die kunnen worden geïnstalleerd via de npm (Node Package Manager). Door gebruik te maken van deze modules, kunnen ontwikkelaars tijd besparen bij het bouwen van applicaties en kunnen ze profiteren van de functionaliteit en het gemak dat deze modules bieden. Hieronder vind je enkele voorbeelden van populaire en nuttige externe modules:

  1. Express - Dit is een van de populairste frameworks voor het bouwen van webapplicaties in Node.js. Het biedt een eenvoudige manier om HTTP-servers te maken, routes te definiëren, middleware te gebruiken enz.
  2. Socket.io - Deze module biedt een eenvoudige manier om real-time communicatie te implementeren tussen de server en de client, bijvoorbeeld voor het bouwen van chatapplicaties.
  3. Moment.js - Dit is een handige module voor het werken met datums en tijden in JavaScript. Het biedt functies om datums te formatteren, te manipuleren en te vergelijken.
  4. Mongoose - Dit is een Object Document Mapper (ODM) voor MongoDB-databases, die het gemakkelijker maakt om met de database te werken en gegevens te valideren.

Dit zijn slechts enkele voorbeelden van de interessante externe modules. Node Package Manager biedt veel meer modules die kunnen worden gebruikt om verschillende taken te automatiseren en de ontwikkeling van applicaties te vereenvoudigen. De volledige lijst aan interne modules kan je vinden op de website van Node Package Manager

Dataopslag

TAGS: Databases, mongodb, postgreSQL

Er zijn verschillende databases die compatibel zijn met Node.js, elk met zijn eigen voordelen en toepassingsgebieden. Enkele van de meest populaire databases voor Node.js zijn:

  • MongoDB: Dit is een documentgeoriënteerde database die gebruikmaakt van JSON-achtige documenten voor gegevensopslag. Het is een uitstekende keuze voor Node.js-toepassingen vanwege de flexibiliteit, schaalbaarheid en snelheid.
  • MySQL: MySQL is een relationele database die gebruikmaakt van SQL (Structured Query Language) voor het opvragen en manipuleren van gegevens. Het is een bewezen oplossing voor zakelijke toepassingen en is geschikt voor zowel kleine als grote projecten.
  • PostgreSQL: PostgreSQL is een geavanceerde open source relationele database die bekend staat om zijn robuustheid, stabiliteit en schaalbaarheid. Het is een uitstekende keuze voor toepassingen met veel gegevens of voor bedrijfskritieke toepassingen.
